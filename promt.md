# Промт для Cursor: редизайн главной страницы litcore.ai

## Контекст

Работаем в проекте на **Astro + Tailwind**. Задача — не переписать сайт с нуля, а
поднять визуальный и интерактивный уровень **главной страницы** до премиального,
взяв лучшие приёмы из трёх референсов и совместив их в единую, ранее
согласованную дизайн-систему проекта ("Stone & Signal" — тёплый минерально-графитовый
минимализм с янтарным сигнальным акцентом, инженерная сетка, tactile brutalism).

Не копируем референсы дословно — вытаскиваем **приёмы**, не чужой брендинг.

## Шаг 0 — сначала аудит, не код

Прежде чем менять код:
1. Пройди по текущей структуре главной страницы (компоненты, секции, layout).
2. Зафиксируй списком: какие секции есть сейчас, из каких компонентов состоят,
   есть ли уже общий слой токенов (цвета/шрифты/spacing) в Tailwind-конфиге.
3. Предложи план: какие секции получают новые эффекты, какие компоненты
   создаются с нуля (например `HeroCore.astro`, `CustomCursor.tsx`,
   `NoiseTextHover.tsx`), какие — просто рефакторятся.
4. Покажи план перед тем, как начать переписывать секции целиком.

## Референсы и что из них берём

**Референс 1 (текущий/ES-вариант litcore, скриншот "Zenrixa"-стиль)**
Нравится: цветовые решения, идея интерактивного "ядра" — элемент, с которым можно
взаимодействовать и как бы "собирать" вокруг него что-то своё (было реализовано
в испанской версии).
Не нравится и НЕ переносим: то, что при смене языка сайт превращается в
полностью другой сайт с другим наполнением и другим опытом. Это баг, а не фича —
языковой переключатель (ES/RU/EN) должен переключать контент, а не дизайн и не
набор фич. Интерактивное "ядро" должно быть **общим компонентом**, доступным на
всех языковых версиях одинаково.

**Референс 2 (Kinetic Studio, скриншот "Desktop Presentation")**
Берём: плавающий/кинетический текст (крупные заголовки, которые слегка живут при
скролле/движении мыши), визуализацию "ядра"/центрального объекта в hero.
Добавляем сверх референса: делаем это ядро интерактивным (реагирует на курсор/скролл/клик),
а не просто декоративной картинкой.

**Референс 3 (litcore-wow.preview.emergentagent.com)**
Берём: необычный кастомный курсор (меняет форму/поведение над разными зонами),
шумовой/grain-эффект при наведении на текст (текстура, не блюр).

## Что делаем конкретно на главной

### 1. Hero-секция
- Тёмный "крепостной" hero: инженерная сетка на фоне + янтарное свечение
  (`--color-accent-bg: #E8A33D`, `--color-accent: #8F5015` — см. токены ниже).
- Центральный интерактивный элемент "ядро" (WebGL/Three.js): реагирует на движение
  курсора (лёгкий parallax-tilt/rotate), можно кликать/тащить — как будто
  пользователь "настраивает" его под себя. Это сквозной брендовый элемент —
  переиспользуемый компонент, не one-off для главной.
  - Lazy-load, poster-фолбэк (статичный кадр) на первом рендере и на мобильных
    с слабым GPU.
  - Обязательно уважает `prefers-reduced-motion` (тогда — статичная картинка/лёгкий fade).
- Кинетический заголовок: крупный дисплейный текст, вариативная ось шрифта
  (wght) слегка "дышит" или реагирует на скролл — без анимации letter-by-letter,
  без "печатной машинки".

### 2. Параллакс — разрешаем шире, не только hero
В отличие от исходного правила блюпринта ("параллакс только в hero"), для
главной страницы разрешаем параллакс-эффекты и в последующих секциях:
- Разная скорость движения слоёв фона/переднего плана при скролле
  (`background-attachment` / `IntersectionObserver` + `transform: translateY`,
  либо CSS `animation-timeline: scroll()` с фолбэком).
- Держим в голове производительность: параллакс — на CSS-transform/GPU-слоях,
  не на repaint-тяжёлых свойствах.
- На мобильных — либо отключаем/упрощаем параллакс, либо делаем его существенно
  мягче (короткая амплитуда), т.к. scroll jank на мобильных особенно заметен.
- `prefers-reduced-motion: reduce` — параллакс отключается везде.

### 3. Кастомный курсor
- Заменяем системный курсор на кастомный (кружок/точка + обводка), который:
  - увеличивается/меняет форму над интерактивными элементами (кнопки, карточки,
    "ядро"),
  - можно сделать в виде mix-blend-mode для контраста на любом фоне.
- Только на десктопе (`@media (pointer: fine)`), на тач-устройствах — обычное
  поведение.

### 4. Grain/noise-эффект при наведении на текст
- При ховере на ключевые заголовки/ссылки — лёгкая шумовая текстура (SVG
  `feTurbulence` или CSS-noise через background-image), не блюр и не глитч.
- Это же можно использовать и как фоновую текстуру карточек/секций (в
  Stone & Signal уже заложены "CSS-текстуры вместо тяжёлого 3D" — зерно/шум как
  замена тяжёлым эффектам).

### 5. Остальные секции (bento, "что автоматизируем", демо-полоса)
- Bento-сетка услуг (карточки A0–A5): асимметричные модули, 1px-рамки
  (`--color-line`), радиус 12px, без теней (или очень мягкая тень 1-2px).
- Цифры/лейблы — моноширинный шрифт (JetBrains Mono), `tabular-nums`.
- Секции чередуют тёмный hero → светлые "известняковые" блоки (`--color-bg: #F6F4EF`).

## Дизайн-токены (уже согласованы, используем как основу)

```css
:root {
  --color-bg: #F6F4EF;
  --color-surface: #FFFFFF;
  --color-ink: #1C2830;
  --color-ink-soft: #46545F;
  --color-line: #DCD6C8;
  --color-accent: #8F5015;
  --color-accent-bg: #E8A33D;
  --color-success: #2F6B4F;
}
:root[data-theme="dark"] {
  --color-bg: #0F151B;
  --color-surface: #161E26;
  --color-ink: #EAE6DC;
  --color-ink-soft: #A7B0B8;
  --color-line: #2B3641;
}
```

Шрифты: **Geologica** (заголовки, variable wght 100–900), **Onest** (текст),
**JetBrains Mono** (лейблы/цифры). Все — с поддержкой кириллицы, self-host через
`@fontsource-variable/*`.

## Чего НЕ делать (анти-паттерны)

- Никаких фиолетово-синих градиентов, неоновых blob-мешей, каши из glassmorphism.
- Никаких эмодзи вместо иконок (единый stroke-набор на базе Lucide).
- Никаких стоковых 3D-роботов/мозгов/"нейросетей из точек".
- Не плодить разный опыт по языковым версиям — "ядро" и все интерактивные
  фичи одинаковы на ES/RU/EN, меняется только текст.
- Не использовать анимацию ради анимации — каждый эффект должен либо направлять
  внимание, либо усиливать премиальность, не отвлекать от текста и CTA.

## Технические требования

- `prefers-reduced-motion` уважается на всех новых эффектах без исключений.
- WebGL/Three.js-компонент — динамический импорт (не блокирует основной бандл),
  постер-фолбэк для мобильных/слабых устройств.
- Кастомный курсор и hover-эффекты — только `@media (pointer: fine)`.
- Тач-таргеты кнопок ≥44×44px, фокус-стили не убираем (`:focus-visible`,
  2px outline `--color-accent-bg`).
- Итог — Lighthouse Performance на десктопе не хуже, чем было до изменений
  (WebGL и тяжёлые эффекты не должны просадить CWV).

## Формат сдачи

Работай по секциям (hero → параллакс-секции → bento → курсор/hover-эффекты как
финальный слой поверх всего). После каждой секции — короткое резюме, что
изменено и какие компоненты созданы/переиспользованы.
